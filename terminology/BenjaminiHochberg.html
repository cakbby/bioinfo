出来ること

多重検定問題に対し、ある程度間違いを許容した緩い検定ができる。ただし、各検定が独立であると仮定する。

False Discovery Rate (FDR)

多重検定問題

複数の検定を繰り返す多重検定では、第一種の過誤 (Type Ⅰ error) が問題となってくる。例えば、一回ごとの検定の有意水準が 1% だったとしても、10000個のサンプルに関して検定を行った場合に、平均して100個のサンプルが（本来は帰無仮説に従うはずが）誤って棄却されてしまう可能性がある。

そこで、誤って棄却されるものの数を減らすために、FWER (Family-Wise Error Rate) という数値に基づいて考えることがある。これは、仮に全てのデータが帰無仮説に従うとして、その中から誤って1つでも棄却してしまう確率である。10000個のサンプルすべてが帰無仮説に従うとして、αFWER=0.01\alpha_{FWER} = 0.01αFWER​=0.01 とすると、検定を100回行ったとしても99回は全てを棄却する。こうすれば、誤って棄却してしまうサンプルの数を減らすことが出来る。

FWERを定めて多重検定を行う方法として Bonferroniの補正 などが挙げられる。しかし、これらの方法はあまりにも厳しい検定となる。例えば先ほどの10000個のサンプルの例では、αFWER=0.01\alpha_{FWER} = 0.01αFWER​=0.01 に対応する各検定の有意水準は α=0.000001\alpha = 0.000001α=0.000001 となってしまう。

ここまで厳しく制限するのではなく、例えば可能性のあるサンプルを多めに取ってくるスクリーニングのような状況でも利用できる方法として、FDR (False Discovery Rate) が提案されている(Benjamini & Hochberg (1995))。

詳細

FDRとは、"棄却されたサンプルの中で実際には帰無仮説に従うものの割合（偽陽性率）の期待値" である。つまり、棄却されたサンプルが100個あったときに、FDRが0.5であれば、平均して50個は誤って棄却してしまったものであると考えられる。

各仮説検定の結果棄却されたサンプルが RRR 個存在したとして、そのうち本当は帰無仮説に従うサンプル（すなわち誤って棄却したサンプル）が VVV 個あったとする。

- True False Sum
採択 - - -
棄却 VVV個 R−VR-VR−V個 RRR個

この時、偽陽性の割合 QQQ を次のように定義する。

Q=VRQ = \frac{V}{R} Q=RV​

ただし、 R=0R=0R=0 (もちろん V=0V=0V=0 でもある) のときは 0 とする。VVV や RRR は確率変数であるため、QQQ も確率変数である。そして、FDR は

FDR=E[Q]FDR = \mathbb{E}[Q] FDR=E[Q]

として定義する。例えば、全てのサンプルで帰無仮説がTrueであったとき、R=0R=0R=0ならばQ=0Q=0Q=0で、R>0R>0R>0ならばQ=1Q=1Q=1となるため、

E[Q]=P(R≥1)\mathbb{E}[Q] = P(R \geq 1) E[Q]=P(R≥1)

となり、これは FWER と等しくなる。したがって、FDR は FWER を内包した表現となっている。

Benjamini-Hochberg法はFDRを指定の値に定めながら棄却するかどうか決めることが出来る方法である。

Python

statsmodels パッケージ の stats.multitest モジュールを用いる。

import statsmodels.stats.multitest

p = np.random(10)
fdr = multitest.multipletests(p, alpha=0.1, method='fdr_bh')
