出来ること

fastqファイルとリファレンス配列から高速にマッピングを行える。

インストール

伝統的な方法

http://daehwankimlab.github.io/hisat2/download/

ここからバイナリを落としてくる（Linux or MacOS）

unzip hisat2-2.2.1-Linux_x86_64.zip

PATHを通す。.bashrc に以下を追加する（~ の直下にhisat2のディレクトリを展開した場合）

export PATH="$PATH:$HOME/hisat2-2.2.1/"

conda を使う方法

bioconda 環境があれば以下のコマンドでOK (brewでも出来る)

conda install -c bioconda hisat2

確認

hisat2 -h

と打ってヘルプが出てくればOK

なお、

hisat2-build -h
# /usr/bin/env: python: No such file or directory

となった場合は、2系のPythonが入ってないので入れる必要がある。

sudo apt-get install python

hisat2-build : インデックス作成

ヒト・マウスなどの有名な種の場合

GRCh38(hg38) や GRCm38(mm10) などはインデックスファイルが作られているので、それを利用する。

http://daehwankimlab.github.io/hisat2/download/

解凍

tar xvf grch38_snptran.tar.gz

インデックスファイルは

~~.*.ht2

の形式をとっており、

genome_snp_tran.1.ht2 genome_snp_tran.2.ht2 … genome_snp_tran.8.ht2

のように8つ出来る。

hisat2 : マッピング

hisat2コマンドによってfastqファイル(fastq.gzでも可)をインデックスによってマッピングする。インデックスの8つの.ht2ファイルはfastqと同じ階層において実行する。

シングルエンドの場合の例

hisat2 -x インデックスの接頭辞 -f -U fastaファイル -S 出力SAMファイル名

ペアエンドの場合の例

hisat2 -x インデックスの接頭辞 -1 1つ目のfastq -2 2つ目のfastq -S 出力SAMファイル名 -p スレッド数

インデックスの接頭辞とは、例えば

genome_snp_tran.*.ht2

のようなインデックスの場合での

genome_snp_tran

の部分を指す。

オプションは以下の通り

****オプション****

-x インデックスの指定
-S 出力するSAMファイルの名前の指定

-U シングルエンドの場合
-f FASTAファイルを使うとき

-1 ペアエンドの場合の1つ目のデータ
-2 ペアエンドの場合の2つ目のデータ

-p スレッド数

例えば test1_1.fastq, test1_2.fastq, test2_1.fastq, ... , test5_2.fastq のようなデータがあるとき、

for i in $(seq 1 5); do \
  hisat2 -x genome -1 test${i}_1.fastq -2 test${i}_2.fastq -p 8 -S test${i}.sam

のように実行すればSAMを5つ作ってくれる。samtools にパイプするなどでBAMへと自動で変換することが出来る。

SRAから直接マッピング

hisat2 -x index --sra-acc accession -S output_sam_name

--sra-acc オプションにSRA系のAccession番号を直接入れても動作する。シングルエンド/ペアエンドかは勝手に判断してくれる。
